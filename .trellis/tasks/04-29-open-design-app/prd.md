# OpenDesign — 产品需求文档 (PRD)

## 一、产品概述

### 1.1 产品愿景

OpenDesign 是一款 AI 驱动的设计+代码一体化工具，融合 Figma Make 的设计生成能力与 Claude Code 的代码编辑体验。用户通过自然语言对话生成 UI 设计，实时预览并可直接编辑代码，实现从概念到生产的完整工作流。

### 1.2 核心定位

**设计 + 代码一体化** — 区别于纯设计工具（Figma、Stitch）和纯代码工具（v0、Lovable），OpenDesign 强调：
- 设计即代码：生成的 UI 是可运行的 React 代码，而非静态设计稿
- 代码即设计：修改代码实时反映到设计预览，双向同步
- 多模型接入：支持 OpenAI 兼容 API、MCP 协议、Ollama 本地模型

### 1.3 目标用户

| 用户群体 | 核心需求 | 技术门槛 |
|---------|---------|---------|
| 产品经理 | 快速原型验证、需求可视化 | 低 |
| 独立开发者 | 加速 UI 开发、减少重复工作 | 中 |
| 设计师 | 探索设计方向、生成设计代码 | 低-中 |
| 设计系统团队 | 保持设计一致性、组件库集成 | 高 |

## 二、核心功能

### 2.1 对话式设计生成

**功能描述**：用户通过自然语言描述设计需求，AI 生成对应的 React 组件代码并实时预览。

**交互流程**：
```
用户输入 → AI 理解意图 → 生成代码 → 实时预览 → 用户反馈迭代
```

**支持的输入方式**：
- 自然语言描述："做一个深色主题的仪表盘"
- 图片上传：截图/设计稿转代码
- 草图绘制：手绘线框图数字化
- 设计系统引用："使用 Material Design 风格"

### 2.2 设计画布

**功能描述**：实时预览生成的 UI，支持可视化编辑和代码编辑双向同步。

**画布架构**：
```
┌─────────────────────────────────────────────────────────────┐
│                      设计画布                                │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              HTML/CSS 渲染层                         │   │
│  │  (React 组件树，支持拖拽、选择、编辑)                 │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Canvas 覆盖层                           │   │
│  │  (绘制选择框、辅助线、缩略图预览)                     │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**画布功能**：
- 实时预览：React 组件热更新渲染
- 元素选择：点击选中组件，高亮显示
- 拖拽调整：支持布局和位置调整
- 响应式预览：切换桌面/平板/手机视图
- 导出代码：一键复制或下载完整代码

### 2.3 代码编辑器

**功能描述**：内置 Monaco Editor，支持直接编辑生成的代码，修改实时同步到画布。

**编辑器功能**：
- 语法高亮：React/TypeScript/CSS
- 智能补全：基于项目上下文
- 错误提示：实时语法检查
- Git 集成：查看代码变更历史

### 2.4 多模型接入

**功能描述**：统一模型适配层，支持多种 AI 模型接入。

**支持的模型类型**：

| 类型 | 接入方式 | 适用场景 |
|------|---------|---------|
| OpenAI 兼容 API | REST API | 云端模型（GPT、Claude、DeepSeek 等） |
| MCP 协议 | MCP Client | 工具扩展、自定义能力 |
| Ollama 本地模型 | REST API | 隐私敏感、离线场景 |

**统一接口设计**：
```typescript
interface ModelAdapter {
  chat(messages: Message[], tools?: Tool[], options?: ChatOptions): Promise<Stream<Response>>;
  stream(callback: (chunk: string) => void): void;
}
```

### 2.5 联网搜索

**功能描述**：集成 Tavily 搜索 API，支持 AI 联网获取最新设计趋势、组件库文档等。

**搜索场景**：
- 设计趋势："2026 年最新的仪表盘设计趋势"
- 组件文档："shadcn/ui 的 DataTable 组件用法"
- 最佳实践："React 性能优化最佳实践"

### 2.6 设计系统集成

**功能描述**：支持导入自定义设计系统组件库，保持设计一致性。

**集成方式**：
- npm 包导入：导入公开或私有 React 组件库
- Token 同步：同步设计 tokens（颜色、字体、间距）
- Guidelines 注入：教导 AI 正确使用组件

## 三、技术架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      前端应用层                              │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │   Chat UI   │ │ Design Canvas│ │ Code Editor │           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    核心服务层                                │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐           │
│  │ Model Layer │ │ Search Layer│ │ Canvas Layer│           │
│  └─────────────┘ └─────────────┘ └─────────────┘           │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    平台适配层                                │
│  ┌─────────────┐ ┌─────────────┐                           │
│  │   Web       │ │ Tauri Desktop│                           │
│  └─────────────┘ └─────────────┘                           │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 技术栈选型

| 层级 | 技术选型 | 理由 |
|------|----------|------|
| **前端框架** | React 18 + TypeScript | 生态最大、类型安全 |
| **UI 库** | Tailwind CSS + shadcn/ui | Claude 风格、高度可定制 |
| **状态管理** | Zustand + TanStack Query | 轻量、服务端状态分离 |
| **跨平台** | Tauri 2.0 | 轻量（3-5MB）、支持移动端 |
| **设计画布** | HTML/CSS + dnd-kit | AI 生成友好 |
| **代码编辑器** | Monaco Editor | VS Code 同款 |
| **模型接入** | Anthropic SDK + MCP SDK | 官方支持 |
| **搜索服务** | Tavily | AI 优化 |
| **后端服务** | Supabase | 认证、数据库、存储 |

## 四、UI 设计规范

### 4.1 配色方案

遵循 Claude.ai 设计风格：

| 用途 | 色值 | 说明 |
|------|------|------|
| 背景主色 | `#f5f5f4` (stone-100) | 暖灰色，柔和 |
| 卡片/面板 | `#ffffff` | 纯白 |
| 主文字 | `#1c1917` (stone-900) | 深灰近黑 |
| 辅助文字 | `#78716c` (stone-500) | 中灰色 |
| 强调色 | `#d97706` (amber-600) | 暖橙琥珀色 |
| 链接 | `#2563eb` (blue-600) | 蓝色 |
| 代码背景 | `#1e1e2e` | 深色（Catppuccin 风格） |
| 边框 | `#e7e5e4` (stone-200) | 浅灰 |

### 4.2 布局模式

```
┌─────────────────────────────────────────────────────────────┐
│  顶栏 (Logo + 模型选择 + 设置)                               │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  侧边栏   │              主内容区                            │
│  - 历史   │                                                  │
│  - 项目   │     ┌─────────────────────────────┐             │
│  - 设置   │     │     对话消息流               │             │
│          │     │     - 用户消息 (右对齐)        │             │
│          │     │     - AI 回复 (左对齐)         │             │
│          │     │     - 代码块 (语法高亮)        │             │
│          │     │     - 工具调用 (折叠展示)      │             │
│          │     └─────────────────────────────┘             │
│          │                                                  │
│          │     ┌─────────────────────────────┐             │
│          │     │     输入框                    │             │
│          │     │     (自动扩展 textarea)       │             │
│          │     │     [附件] [发送]             │             │
│          │     └─────────────────────────────┘             │
├──────────┴──────────────────────────────────────────────────┤
│  状态栏 (模型信息 + Token 用量)                               │
└─────────────────────────────────────────────────────────────┘
```

### 4.3 交互特点

- **消息气泡**：用户消息右对齐，AI 回复左对齐，无头像
- **代码块**：深色背景，语法高亮，支持一键复制
- **工具调用**：折叠式展示，显示工具名、输入参数、执行结果
- **流式输出**：逐字显示，带闪烁光标效果
- **输入框**：底部固定，自动扩展高度，Shift+Enter 换行

## 五、开发阶段规划

### Phase 1: 基础架构 (P0)

**目标**：搭建项目骨架，实现基础对话界面

**交付物**：
- [ ] Next.js 项目初始化（SSG 模式）
- [ ] Tailwind CSS + shadcn/ui 配置
- [ ] 基础布局组件（顶栏、侧边栏、主内容区）
- [ ] 对话界面组件（消息流、输入框）
- [ ] Zustand 状态管理配置

### Phase 2: 模型接入层 (P0)

**目标**：实现多模型统一接入

**交付物**：
- [ ] 统一消息格式定义
- [ ] OpenAI API 适配器
- [ ] MCP Client 适配器
- [ ] Ollama 适配器
- [ ] 模型选择 UI
- [ ] 流式响应处理

### Phase 3: 设计画布 + 代码编辑器 (P0)

**目标**：实现核心编辑能力

**交付物**：
- [ ] HTML/CSS 渲染画布
- [ ] Canvas 覆盖层（选择框、辅助线）
- [ ] dnd-kit 拖拽集成
- [ ] Monaco Editor 集成
- [ ] 代码-画布双向同步
- [ ] 响应式预览切换

### Phase 4: 搜索集成 (P1)

**目标**：实现联网搜索能力

**交付物**：
- [ ] Tavily API 集成
- [ ] 搜索结果注入对话
- [ ] 来源引用展示

### Phase 5: Tauri 桌面端 (P1)

**目标**：打包为桌面应用

**交付物**：
- [ ] Tauri 项目配置
- [ ] 原生窗口集成

### Phase 6: 用户系统 (P2)

**目标**：实现用户认证和数据同步

**交付物**：
- [ ] Supabase 认证集成

## 六、成功指标

### 6.1 功能指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 代码生成准确率 | ≥85% | 人工评审 |
| 首屏加载时间 | ≤2s | Lighthouse |
| 流式响应延迟 | ≤500ms | 首字显示时间 |
| 画布渲染帧率 | ≥30fps | 性能监控 |

### 6.2 用户体验指标

| 指标 | 目标值 | 测量方式 |
|------|--------|---------|
| 任务完成率 | ≥80% | 用户测试 |
| 用户满意度 | ≥4.0/5.0 | NPS 调研 |
| 学习曲线 | ≤30min | 新用户上手时间 |

## 七、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| AI 生成代码质量不稳定 | 高 | 引入代码审查机制，支持人工修正 |
| 多模型 API 兼容性问题 | 中 | 统一适配层，充分测试 |
| Tauri 跨平台兼容性 | 中 | 优先 Web 版本，渐进式桌面端 |
| 设计系统导入复杂度 | 中 | 先支持主流组件库，逐步扩展 |
